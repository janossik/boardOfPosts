Dokumentacja

Api ma na celu obsługe zarządzania postami w aplikacjach działających jak tablice ogłoszeń. Celem pracy była prostota i łatwość korzystania z tego rozwiązania.

Projekt wykorzystuje następujące pakiety:
  • qs
  • lodash
  • dotenv
  • express
  • passport
  • jsonwebtoken
  • mongoose
  • passport-jwt
  • passport-local
  • mongoose-url-slugs
  • mongo-cursor-pagination
  • passport-local-mongoose

Autorami tego api są Marcin Czaniecki i Agata Karczewska

Dokumentacja api

Typ: get

endpoint:

/

Pobierz wszystkie posty

Typ: get

endpoint:

/posts

Query

limit - Ile elementów będziemy pobierać. Domyślnie ustawiony na 2 (Zapytanie zwróci 2 elementy)

Zwracany nagłówki (Headers)

next - link z zapytaniem do serwera o kolejne 2 elementy

previous - link z zapytaniem do serwera o poprzednie 2 elementy

Pobierz wszystkie posty posiadające odpowienie query

Typ: get

endpoint:

/posts/search

Query

offset - Od którego obiektu zaczynamy pobierać elementy. Domyślnie ustawiony na 0

per_page - Ile elementów będziemy pobierać. Domyślnie ustawiony na 2 (Zapytanie zwróci 2 elementy)

sort_by - Sortuj według tego właściwości elementów. Domyślnie ustawiony na createdAt

order_by - W jaki sposób ma być sortowane. Domyślnie ustawiony na desc

Pobierz jeden post

Typ: get

endpoint:

/posts/:slug

Dodaj post

Typ: POST

endpoint:

/posts

Edytuj post

Typ: PUT

endpoint:

/posts/:slug

Usuń post

Typ: DELETE

endpoint:

/posts/:slug

Nieoczywiste

slug - przyjazna unikatowa nazwa elementu, jeżeli nie jest znana można wpisać id elementu.

Struktura posta

Post jest skonstruowany za pomocą mongoose schema i wyeksportowany jako model

Autoryzacja użytkownika

Do autoryzacji użytkownika wykorzystujemy middleware passport jest to zestaw wtyczej (strategie), które umożliwają utworzenie prostej autoryzacji w kodzie aplikacji.
